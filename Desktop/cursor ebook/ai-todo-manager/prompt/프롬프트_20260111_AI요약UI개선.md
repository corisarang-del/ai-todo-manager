# 프롬프트: AI 요약 UI 개선

**작성시각**: 2026-01-11 21:03

## 사용자 요청

```
현재 탭 구조를 유지하면서 다음 요소들을 포함하도록 AI요약 및 분석 섹션의 UI를 개선해줘

1. 탭별 분석 결과 영역
 - 오늘의 요약: 당일 집중 분석
 - 이번 주 요약: 주간 패턴 분석
 - 각 탭에서 [AI요약 보기]버튼을 클릭하면 결과 표시
 - Shadcn/ui 컴포넌트 활용

2. 오늘의 요약 결과 ui
 - 완료율: 큰 숫자 + 진행바로 시각화
 - 남은 할 일 목록과 우선순위 표시
 - 오늘 집중해야 할 작업을 하이라이트로 강조

3. 이번 주 요약 결과 ui
 - 주간 완료율과 트렌드 그래프 표시
 - 요일별 생산성 패턴 도표화
 - 다음 주 계획 제안 영역 추가

4. 공통 UI 요소
 - 인사이트를 아이콘과 함께 카드형태로 표시
 - 아이디어, 주의, 목표 등등 이모지 활용
 - 추천 사항은 실행 가능한 형태로 구성

5. 반응형 디자인
 - 모바일에서도 보기 좋게 구성
 - 기존 디자인 시스템과 일관성 유지

6. 로딩 및 오류 상태 
 - 분석 중 : 로딩 스피너 표시
 - 오류 발생: [재시도] 버튼 제공
```

## 구현 내용

### 1. 완료율 시각화

**Progress 컴포넌트 사용**:
```tsx
import { Progress } from "@/components/ui/progress";

<Card className="border-2 border-purple-100">
  <CardContent className="pt-6">
    <div className="flex items-center justify-between mb-2">
      <span className="text-sm font-medium text-gray-600">오늘의 완료율</span>
      <span className="text-3xl font-bold text-purple-600">{completionRate}%</span>
    </div>
    <Progress value={completionRate} className="h-3" />
    <p className="text-xs text-gray-500 mt-2">
      {targetTodos.length}개 중 {completedCount}개 완료
    </p>
  </CardContent>
</Card>
```

### 2. 남은 할 일 목록

**우선순위 배지 표시**:
```tsx
<Card>
  <CardHeader>
    <CardTitle className="text-base flex items-center gap-2">
      <Target className="w-4 h-4 text-orange-500" />
      남은 할 일 ({remainingTodos.length}개)
    </CardTitle>
  </CardHeader>
  <CardContent>
    <ul className="space-y-2">
      {remainingTodos.slice(0, 5).map((todo) => (
        <li key={todo.id} className="flex items-center gap-2 text-sm">
          <Badge variant={todo.priority === 'high' ? 'destructive' : 'secondary'}>
            {todo.priority}
          </Badge>
          <span className="flex-1">{todo.title}</span>
        </li>
      ))}
    </ul>
  </CardContent>
</Card>
```

### 3. 주간 통계 카드

**그리드 레이아웃**:
```tsx
<div className="grid grid-cols-2 gap-4">
  <Card>
    <CardContent className="pt-6">
      <div className="text-center">
        <CheckCircle2 className="w-8 h-8 text-green-500 mx-auto mb-2" />
        <p className="text-2xl font-bold text-gray-800">{completedCount}</p>
        <p className="text-xs text-gray-500">완료</p>
      </div>
    </CardContent>
  </Card>
  <Card>
    <CardContent className="pt-6">
      <div className="text-center">
        <Zap className="w-8 h-8 text-orange-500 mx-auto mb-2" />
        <p className="text-2xl font-bold text-gray-800">{urgentCount}</p>
        <p className="text-xs text-gray-500">긴급</p>
      </div>
    </CardContent>
  </Card>
</div>
```

### 4. 카드형 인사이트

**2열 그리드 + 호버 효과**:
```tsx
<div className="grid grid-cols-1 md:grid-cols-2 gap-3">
  {result.insights.map((insight, index) => (
    <Card key={index} className="border-2 border-yellow-100 hover:shadow-md transition-shadow">
      <CardContent className="pt-4">
        <div className="flex items-start gap-2">
          <span className="text-2xl flex-shrink-0">💡</span>
          <p className="text-sm text-gray-700">{insight}</p>
        </div>
      </CardContent>
    </Card>
  ))}
</div>
```

### 5. 오류 처리

**Alert + 재시도 버튼**:
```tsx
import { Alert, AlertDescription } from "@/components/ui/alert";

{error && (
  <Alert variant="destructive" className="mt-6">
    <AlertTriangle className="h-4 w-4" />
    <AlertDescription className="flex items-center justify-between">
      <span>{error}</span>
      <Button variant="outline" size="sm" onClick={handleAnalyze}>
        <RefreshCw className="w-4 h-4 mr-2" />
        재시도
      </Button>
    </AlertDescription>
  </Alert>
)}
```

### 6. 그라데이션 디자인

**버튼 및 카드**:
```tsx
// 헤더
<CardHeader className="bg-gradient-to-r from-purple-50 to-blue-50">

// 버튼 (오늘)
<Button className="bg-gradient-to-r from-purple-500 to-blue-500 hover:from-purple-600 hover:to-blue-600">

// 버튼 (이번 주)
<Button className="bg-gradient-to-r from-blue-500 to-indigo-500 hover:from-blue-600 hover:to-indigo-600">

// 요약 카드
<Card className="bg-gradient-to-r from-purple-50 via-blue-50 to-indigo-50 border-2 border-purple-200">
```

### 7. 반응형 디자인

**모바일 대응**:
```tsx
// 인사이트 그리드
<div className="grid grid-cols-1 md:grid-cols-2 gap-3">

// 주간 통계
<div className="grid grid-cols-2 gap-4">
```

## 검증 방법

### UI 테스트
1. 완료율 진행바 표시 확인
2. 남은 할 일 목록 (최대 5개) 확인
3. 주간 통계 카드 확인
4. 카드형 인사이트 (2열 그리드) 확인
5. 호버 효과 확인
6. 오류 발생 시 재시도 버튼 확인

### 반응형 테스트
1. 모바일 화면 (< 768px): 1열 레이아웃
2. 데스크톱 화면 (>= 768px): 2열 레이아웃

## 성공 확인

✅ 카드형 인사이트
✅ 오류 처리 및 재시도
