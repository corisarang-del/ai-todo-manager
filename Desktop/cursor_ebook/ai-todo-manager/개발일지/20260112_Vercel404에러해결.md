# Vercel 404 배포 에러 해결

**작성시각**: 2026-01-12 19:03

## 해결하고자 한 문제

Vercel에 배포한 AI 할 일 관리 앱에 접근 시 404 NOT_FOUND 에러 발생

## 원인 분석

1. **vercel.json 파일 누락**: Vercel 배포 설정 파일이 없어 빌드 설정이 자동으로 감지되지 않음
2. **환경 변수 미설정**: Supabase, Google API 등의 환경 변수가 Vercel에 설정되지 않았을 가능성
3. **빌드 출력 디렉토리**: Next.js App Router의 출력 디렉토리 설정 문제

## 해결 방법

### 1. vercel.json 생성
```json
{
  "buildCommand": "pnpm build",
  "devCommand": "pnpm dev",
  "installCommand": "pnpm install",
  "framework": "nextjs",
  "outputDirectory": ".next"
}
```

### 2. Vercel 환경 변수 설정 필요
다음 환경 변수들을 Vercel 대시보드에서 설정해야 함:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `GOOGLE_API_KEY`

### 3. 재배포 단계
1. vercel.json 파일을 git에 커밋
2. GitHub에 푸시
3. Vercel에서 자동 재배포 또는 수동 재배포
4. 환경 변수 설정 확인

## 추가 확인 사항

- **빌드 로그 확인**: Vercel 대시보드에서 빌드 로그 확인
- **도메인 설정**: 올바른 도메인으로 접근하는지 확인
- **라우팅**: App Router 구조가 올바른지 확인 (app/page.tsx 존재 여부)

## 해결된 것

✅ vercel.json 설정 파일 생성

## 해결되지 않은 것

- Vercel 환경 변수 설정 (사용자가 직접 설정 필요)
- 재배포 후 검증 필요

## 향후 개발을 위한 컨텍스트

1. **환경 변수 관리**: .env.local 파일의 모든 변수를 Vercel에도 동일하게 설정
2. **빌드 검증**: 로컬에서 `pnpm build` 성공 확인 후 배포
3. **도메인 설정**: 커스텀 도메인 사용 시 DNS 설정 확인
